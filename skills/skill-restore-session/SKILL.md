# Skill: 恢复工作状态

## 触发条件

- 新对话开始时（上下文为空）
- 用户说"继续上次的工作" / "恢复上下文" / "接着做"
- 用户执行了 `/restore` 或类似恢复命令
- Claude Code 对话因上下文溢出重新开始

## 前置检查

1. **确认当前目录** — 必须在 `/Users/mac-cj/manus/xilian-platform-app/`
2. **确认 git 可用** — `git status` 不报错
3. **确认三重记忆文件存在**:
   - `CLAUDE.md`
   - `docs/daily/` 目录
   - `docs/LESSONS_LEARNED.md`

## 标准步骤

### Step 1: 读取永久记忆 (30s)

```bash
# CLAUDE.md 始终自动加载，重点关注：
# §8 ADR-001~005 — 不可违反的架构决策
# §9 七条设计原则
# §12 自定义命令
```

**关键记忆点**:
- ADR-001: 物理约束优先
- ADR-002: 8 域通过 EventBus 解耦
- ADR-003: 感知三层不可跳层
- ADR-004: 4 段式统一编码
- ADR-005: 质量评分 = 完整度×0.4 + 准确度×0.6

### Step 2: 读取最近的每日记忆 (1min)

```bash
# 找到最近的日志
ls -la docs/daily/*.md | tail -5

# 读取最新日志，重点关注：
# "未完成任务" — 这些是要继续的工作
# "明天的起点" — 直接的下一步指引
# "上下文快照" — 分支、端口、降级状态
```

**从每日日志中提取**:
- 当前分支名和最新提交
- 未完成任务列表
- 开发服务器端口（通常 3002）
- 降级组件（ClickHouse/Kafka 在开发环境预期降级）

### Step 3: 读取经验库 (30s)

```bash
# 浏览 docs/LESSONS_LEARNED.md
# 关注最近添加的条目（按日期排序）
# 特别注意与当前任务相关的经验
```

**关键经验**:
- ARCH-001: 新增不修改原则
- ARCH-002: 单例+get/reset 工厂模式
- DATA-002: 长缺口不插值
- DEBUG-001: 端口 3000 占用时用 3002

### Step 4: 检查 Git 状态 (30s)

```bash
git branch --show-current
git log --oneline -5
git status
```

**从 git 中提取**:
- 当前分支和偏离主干的程度
- 最近的提交信息（了解上次做了什么）
- 未提交的变更（工作进行中的文件）

### Step 5: 读取 Auto-Memory (15s)

```bash
# Claude Code 自动记忆
cat ~/.claude/projects/-Users-mac-cj-manus-xilian-platform-app/memory/MEMORY.md
```

### Step 6: 检查开发总纲 (可选, 2min)

```bash
# 如果需要了解整体进度
head -100 docs/MASTER_DEVELOPMENT_PLAN.md
```

### Step 7: 输出恢复摘要

```markdown
## 上下文恢复完成

**分支**: feature/xxx
**最近提交**: abc1234 message
**未完成任务**:
1. 任务 A — 阻塞原因
2. 任务 B — 待开始

**开发环境**:
- 端口: 3002
- 降级组件: ClickHouse, Kafka (预期)

**建议下一步**:
> 继续 [任务 A]，从 [具体文件/函数] 开始
```

## 必须满足的验收标准

- [ ] 已读取最新每日日志的"未完成任务"
- [ ] 已确认当前分支和 git 状态
- [ ] 已知晓关键 ADR（物理约束优先等）
- [ ] 已输出恢复摘要（分支+未完成+下一步）
- [ ] 未遗漏 LESSONS_LEARNED 中的关键经验

## 恢复层级

| 层级 | 耗时 | 读取内容 | 适用场景 |
|------|------|---------|---------|
| **快速** | <2min | Step 1 + 2 + 4 | 短暂中断，继续同一任务 |
| **标准** | ~5min | Step 1-5 | 新对话开始，恢复上下文 |
| **深度** | ~10min | Step 1-7 | 长时间中断，需要全面回顾 |

## 常见错误和预防

| 错误 | 后果 | 预防 |
|------|------|------|
| 不看每日日志直接开始 | 重复已完成的工作 | 第一步必看"未完成任务" |
| 忘记 ADR | 写出违反架构决策的代码 | 每次恢复重温 5 条 ADR |
| 不检查 git status | 覆盖未提交的修改 | 恢复时必看 git status |
| 忽略 LESSONS_LEARNED | 重复踩坑 | 浏览最近的经验条目 |
| 不看降级状态 | 误以为外部服务故障 | ClickHouse/Kafka 开发环境预期降级 |
