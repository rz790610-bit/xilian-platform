# Phase 2 意见2.0 要点归类

## 意见1（深度评审，8页）

### 一、量化目标与实现手段不匹配（3处）
1. 假设命中率≥88%可信度存疑：短路不产生新假设，冷启动期种子数据不足
   → 建议：分阶段命中率目标（冷启动≥60%，500次会话后≥88%）+ 评测数据集
2. Grok调用率<15%过度依赖短路：港口设备几乎都是高重要度，CostGate自然通过率可能远高于15%
   → 建议：用历史告警数据压力测试CostGate公式，反向校准threshold
3. 物理验证率≥95%与mapping_skip_rate>0.5告警阈值矛盾：分母定义不清
   → 建议：明确physics_verification_rate计算口径

### 二、技术设计深层问题（4处）
4. PINN工程化路径不清晰：PINN_loss依赖训练好的神经网络，不是解析式评估
   → 建议：先用WorldModel方程残差代理（解析式），真PINN推迟到v3.1
5. 三维衰减冷启动伪精度：经验池<50条时工况相似度随机性>信号性
   → 建议：自适应降维（<50条→单维时间衰减，50-200→二维，>200→三维）
6. doIntervention()未观测混淆变量风险：港口机械存在大量不可直接观测的内部状态
   → 建议：内置可识别性检验，检测到未观测混淆变量时返回警告
7. Shadow Mode晋升标准过于简单：仅凭50次会话无显著性检验
   → 建议：至少100次+命中率高5pp+p-value<0.05+P95延迟不超120%

### 三、工程实施注意点（3处）
8. 35小时工时低估：缺少单元测试工时
   → 建议：新增P2-13单元测试（6-8小时）
9. 种子因果图静态表格风险：6条链未经专家验证
   → 建议：P2-3阶段安排领域专家评审会
10. 前端可视化时序风险：Shadow Mode未验证前暴露实验性结果
    → 建议：前端标注来源路径（champion/challenger）+"实验模式"水印

### 四、追加技术方向
11. 异常推理不确定性传播：uncertainty应贯穿整个推理链，最终输出分布而非点估计
    → 建议：因果路径置信度区间+经验匹配相似度分布+最终假设置信度分布

### 总结优先级
1. 分阶段指标目标（预期管理）
2. 种子数据专家验证（基础数据质量）
3. 单元测试工时（复杂逻辑正确性）
4. PINN落地路径预研（2-3小时）
