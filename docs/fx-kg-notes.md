# FX知识图谱1.0 文档核心要点

## 整体架构：混合KG引擎（边缘层 + 云层）
- 边缘层（诊断）：实时图，传感器数据，低延迟<100ms
- 云层（训练推理）：动态知识库图，聚合fleet数据，多跳推理
- 数据流：传感器/IoT → 边缘KG更新 → 云KG融合 → 训练/推理输出 → OTA反馈

## 诊断领域（FSD逻辑）
1. 感知输入：IoT数据 → NER实体提取
2. 实时KG构建：节点=设备/事件，边=动态关系（振动>阈值→风险高→规划减速）
3. 规划融合：GNN遍历图 → 输出本能行动（路径优化、应急停机）
4. 边缘执行：本地计算，fleet共享更新KG

## 训练推理领域（Grok逻辑）
1. 事实grounding：日志→LLM提取→注入KG
2. 关系推断：多跳遍历（GNN+RLHF），自纠错
3. 深度推理：输出可追溯方案
4. 动态更新：实时事件注入

## 用户核心理解
知识图谱 = 数据 + 历史 + 机理 → 图谱化框架 → 快速复用 + 全面诊断
- 不是单纯节点关系可视化
- 是可复用的诊断决策框架
- 基于场景的模板，用于智能诊断/边缘诊断/端到端快速诊断
- 结合自进化模块：自升级、自补充数据维度、迭代
